Частина Б · Вказівники та арифметика

Створіть окремий файл-пісочницю (main.c нового проєкту або окрема функція demo_hw(void)), де виконуватимете задачі Б. Усе друкуйте через ESP_LOGI. Нагадування: &x — «взяти адресу», *p — «значення за адресою», %p — друк адреси.
Розминка

Б1. Оголосіть int x = 42; і int *p = &x;. Надрукуйте чотири рядки: x, &x, p, *p. Які два з них однакові й чому? Потім виконайте *p = 100; і надрукуйте x. Поясніть, чому x змінився, хоча ми не чіпали його напряму.

Б2. Оголосіть int a = 5, b = 9; та два вказівники на них. Поміняйте місцями ЗНАЧЕННЯ a і b, працюючи ТІЛЬКИ через вказівники (не звертаючись до a/b за іменами). Надрукуйте до і після.
Арифметика вказівників

Б3. Оголосіть int arr[5] = {10, 20, 30, 40, 50};. Надрукуйте адреси &arr[0] … &arr[4]. На скільки байтів відрізняються сусідні? Звідки саме це число?

Б4. Для того ж масиву перевірте й поясніть три рівності (надрукуйте обидві сторони):

arr та &arr[0] — однакові?
*(arr + 2) та arr[2] — однакові?
(arr + 4) - (arr + 1) — чому дорівнює? Це байти чи елементи?

Б5. Порахуйте суму елементів arr ДВІЧІ: спершу класичним циклом з індексом arr[i], потім — проходом вказівником: for (int *q = arr; q < arr + 5; q++) sum += *q;. Переконайтесь, що результат однаковий.
Вказівники, масиви та рядки

Б6. Оголосіть рядок char s[] = "ESP32-S3";. Не використовуючи strlen, порахуйте його довжину самі: ідіть вказівником від початку, доки не натрапите на символ '\0'. Окремо надрукуйте s і &s[0] — поясніть, чому вони рівні.

Б7. Напишіть функцію void fill(int *buf, int n, int value), яка заповнює n елементів масиву значенням value, звертаючись до елементів ЧЕРЕЗ вказівник (*(buf + i) або *buf++). Перевірте на масиві з частини Б3.
Вказівники як параметри (повернення «назовні»)

Б8. Напишіть void get_min_max(const int *arr, int n, int *min, int *max), яка через вихідні вказівники min і max повертає мінімум і максимум масиву. Поясніть, навіщо тут const у першого аргументу і чому решта — звичайні вказівники.
Вказівники на структури

Б9. Оголосіть struct point { int x; int y; }; і масив struct point pts[3]. Заповніть його, проходячи вказівником struct point *p = pts; і звертаючись через стрілку p-%3Ex, p->y. Надрукуйте &pts[0] і &pts[1] — на скільки байтів вони відрізняються і чому саме на стільки (підказка: sizeof(struct point))?
